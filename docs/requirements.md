# Requirements Specification Summary

이 문서는 주요 요구사항을 카테고리와 기능별로 요약한 내용입니다.

## 화면
- **[REQ-DIS-001] 진입점 > 메인 랜딩** (기능, 우선순위: 중간)
  - 요약: 모든 비로그인 사용자 진입 시 좌측 상단에 프로젝트 로고 표시 및 스크롤 네비게이션 제공.
- **[REQ-DIS-002] 진입점 > 메인 랜딩** (비기능, 우선순위: 높음)
  - 요약: 랜딩 페이지 하단에 법적 고지 안내 및 개인정보처리안내 링크 필수 노출.
- **[REQ-DIS-003] 진입점 > 메인 랜딩** (비기능, 우선순위: 높음)
  - 요약: 랜딩 페이지 하단에 고객 지원 항목(전화, 이메일) 노출.
- **[REQ-COM-001] 공통 > UI/UX** (기능, 우선순위: 중간)
  - 요약: 고령층을 배려하여 글로벌 헤더 상단에 글자 크기 모드 토글 제공.

## 회원관리
- **[REQ-COM-002] 회원가입 > 가입분기** (기능, 우선순위: 높음)
  - 요약: 초기 진입 시 사용자/보호자 목적에 따라 역할을 분리하여 회원가입 진행.
- **[REQ-USR-001] 회원가입 > 공통 항목** (기능, 우선순위: 높음)
  - 요약: 이메일, 비밀번호, 이름, 성별, 생년월일, 휴대폰 번호 등 기본 폼 입력.
- **[REQ-USR-004] 회원가입 > 공통 항목** (기능, 우선순위: 높음)
  - 요약: 본인 소유 이메일 확인을 위한 이메일 인증코드 전송 및 검증.
- **[REQ-USR-005] 회원가입 > 공통 항목** (기능, 우선순위: 높음)
  - 요약: 회원가입을 위한 휴대폰 인증 기능(5분 유효시간) 제공.
- **[REQ-USR-008] 로그인 > 공통 항목** (기능, 우선순위: 높음)
  - 요약: 이메일과 비밀번호를 통한 로그인 제공.
- **[REQ-USR-014] 비밀번호 찾기 > 비밀번호 재설정** (기능, 우선순위: 높음)
  - 요약: 임시 비밀번호로 로그인한 사용자의 새 비밀번호 설정 제공.
- **[REQ-USR-020] 보호자 연동 > 사용자 등록** (기능, 우선순위: 높음)
  - 요약: 보호자는 사용자 검색을 통해 연동 요청 및 환자의 승인을 통해 매핑.
- **[REQ-USR-027] 로그인 > 소셜 로그인** (기능, 우선순위: 높음)
  - 요약: 카카오 및 구글 계정을 이용한 간편 회원가입 및 로그인 기능 제공.

## 실시간 챗봇
- **[REQ-CHAT-001] 챗봇 UI > 채팅 인터페이스** (기능, 우선순위: 높음)
  - 요약: 가이드 생성 뷰 하단 또는 플로팅 버튼을 통해 챗봇 세션 진입.
- **[REQ-CHAT-002] 사용자 편의 > 액션 칩 및 STT** (기능, 우선순위: 높음)
  - 요약: 고령자 배려를 위한 퀵 액션 칩(버튼) 제공 및 음성 인식(STT) 지원.
- **[REQ-CHT-001] 백엔드 모델 > 맥락 유지 추론** (비기능, 우선순위: 높음)
  - 요약: 처방전 데이터와 직전 질문 로그를 지속적으로 LLM에 전달하여 맥락 유지.
- **[REQ-CHT-002] 스트리밍 UI > 비동기 응답 처리** (비기능, 우선순위: 높음)
  - 요약: LLM 응답 시 실시간 스트리밍 형태로 텍스트 출력.

## AI 가이드
- **[REQ-LLM-001] 정보 통합 > 프롬프트 구성** (비기능, 우선순위: 높음)
  - 요약: OCR 추출 데이터와 사용자 프로필(기저질환, 알러지) 결합으로 맞춤형 프롬프트 생성.
- **[REQ-LLM-002] 생성 제어 > 파라미터 제약** (비기능, 우선순위: 높음)
  - 요약: Hallucination 방지를 위한 Temperature 등의 LLM 파라미터 튜닝.
- **[REQ-LLM-004] 결과물 고지 > 법적 면책 조항** (기능, 우선순위: 높음)
  - 요약: AI가 생성한 가이드에 대해 의료법 및 인공지능 기본법상 책임 면책 조항 명시.

## 복약가이드
- **[REQ-GUIDE-001] 가이드생성 > LLM 연동** (기능, 우선순위: 높음)
  - 요약: 확정된 처방전 데이터를 기반으로 LLM 호출하여 맞춤형 복약 가이드 생성.
- **[REQ-GUIDE-003] 상세조회 > 약물별 상세** (기능, 우선순위: 높음)
  - 요약: 각 약물 구체적 복용법 카드 형태 표시 및 병용 통합 주의사항 강조.
- **[REQ-GUIDE-006] 상세조회 > 생활습관 가이드** (기능, 우선순위: 중간)
  - 요약: 처방 약물과 질환에 맞는 식이요법 및 운동 가이드 추가 제공.

## 대시보드
- **[REQ-DASH-001] 메인화면 > 주요기능** (기능, 우선순위: 높음)
  - 요약: '처방전 업로드', '복약 가이드 확인', 'AI 상담' 카드형 UI 제공.
- **[REQ-DASH-002] 메인화면 > 오늘의 복약 일정** (기능, 우선순위: 높음)
  - 요약: 오늘 복약 일정을 카드 형태로 확인 및 서버 투약 일정 데이터 동기화.
- **[REQ-DASH-006] 사이드바 > 네비게이션** (기능, 우선순위: 중간)
  - 요약: 이용자 Role(환자, 보호자)별 표시 메뉴 분리.

## 업로드 및 OCR 연동
- **[REQ-OCR-001] 처방전 업로드 > 폼 제출 및 안내** (기능, 우선순위: 높음)
  - 요약: 처방전 이미지 업로드(디바이스 파일 첨부 및 카메라 촬영 지원).
- **[REQ-OCR-004] 텍스트 추출 > 외부 API 호출** (기능, 우선순위: 높음)
  - 요약: Naver Clova 또는 Upstage 등의 외부 상용 OCR을 활용해 텍스트 추출.
- **[REQ-OCR-006] 텍스트 추출 > 데이터 정형화** (기능, 우선순위: 높음)
  - 요약: 반환된 Raw 텍스트에서 Key-Value JSON으로 핵심 엔티티 정형화.
- **[REQ-OCR-008] OCR 결과 수정/확정** (기능, 우선순위: 중간)
  - 요약: 잘못 인식된 약물명이나 용량을 사용자가 직접 수정/제거 가능.
